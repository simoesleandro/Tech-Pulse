📐 System Architecture (System Design)

O design do sistema adota um modelo assíncrono e desacoplado para evitar que a latência da inferência de IA local (Gemma via Ollama) degrade a experiência de navegação do usuário no Front-end.

1. Fluxo de Dados de Ingestão (Background Loop)

Etapa 1: Ingestão Ativa (Scraper Engine)

O módulo do FastAPI executa rotinas de captura consumindo feeds e APIs públicas do Dev.to, Reddit e GitHub Trends.

Etapa 2: Sanitização e Deduplicação

Os payloads brutos são limpos. Links já existentes no banco de dados SQLite são descartados imediatamente para poupar hardware.

Etapa 3: Governança por IA Local (Ollama)

O título do artigo é submetido ao modelo Gemma Local. A IA atua como um firewall cognitivo estrito.

Etapa 4: Persistência Estruturada

Artigos validados recebem a flag de relevância e são armazenados fisicamente no arquivo do banco SQLite.

2. Fluxo de Consumo (Client Interface)

O Front-end desenvolvido em React / Next.js realiza chamadas HTTP REST limpas para a API do FastAPI. Como a inteligência e o processamento de dados ocorreram na ingestão, as consultas ao banco de dados SQLite retornam em milissegundos.

🛠️ Technical Stack (Tecnologias)

Camada Tecnologia Escolhida Justificativa Técnica
Front-end React / Next.js + Tailwind CSS Componentização eficiente, renderização rápida e estilização focada em UI limpa e sóbria.
Back-end API FastAPI (Python) Alta performance nativa assíncrona, facilidade de manipulação de dados e documentação automática Swagger.
AI Inference Gemma Local via Ollama Privacidade total dos dados de leitura, custo zero de infraestrutura de nuvem externa (Tokens) e controle de latência.
Database SQLite Serverless, leve, ideal para persistência local em ambientes de desenvolvimento rápidos e isolados.

🗄️ Relational Data Model (Schema SQL)

Abaixo está o esquema DDL estrito da tabela principal que residirá no SQLite, otimizada com indexação de segurança.

CREATE TABLE news_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    url TEXT UNIQUE NOT NULL,          -- Impede duplicidade de links em nível de banco
    source TEXT NOT NULL,              -- 'github_trends', 'dev.to', 'reddit'
    ai_relevance TEXT NOT NULL,        -- 'RELEVANTE' ou 'LIXO'
    is_read BOOLEAN DEFAULT 0,         -- Estado de consumo do usuário
    is_bookmarked BOOLEAN DEFAULT 0,   -- Sistema de salvamento interno
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Índice para acelerar a renderização da fila de notícias não lidas
CREATE INDEX idx_news_unread ON news_items (is_read, ai_relevance);
